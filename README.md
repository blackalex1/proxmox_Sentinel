**PVE Aegis** — это передовое комплексное решение для активной кибербезопасности и интеллектуального мониторинга гипервизора **Proxmox VE** и его **LXC-контейнеров** непосредственно через Telegram. 

В отличие от пассивных систем сбора метрик, **PVE Aegis** содержит полноценный **движок активного противодействия угрозам (IPS)**, который на аппаратном уровне ядра перехватывает подозрительную сетевую активность и мгновенно уничтожает вредоносные процессы.

---

## 🌟 Ключевые возможности системы

### 1. ⚔️ Активная защита и авто-килл (Active IPS Engine)
* **Мгновенное уничтожение угроз (`kill -9`)**: При обнаружении несанкционированного исходящего соединения на чувствительные порты (SSH `22`, RDP `3389`, PostgreSQL `5432`, MySQL `3306`, MongoDB `27017`, PVE Web `8006`):
  * Бот мгновенно перехватывает пакет рукопожатия (TCP SYN) на уровне сетевого моста хоста.
  * Сканирует активные сокеты внутри скомпрометированного LXC-контейнера (`pct exec -- ss -atnup`).
  * Определяет точный PID и имя процесса нарушителя (например, `hydra`, `nmap` или бэкдор-скрипт).
  * **Принудительно уничтожает его (`kill -9`) за 30–50 миллисекунд**, предотвращая атаку до отправки первого байта.
* **Локальный белый список (Self-API Whitelist)**: Интеллектуальное исключение для внутренних служебных запросов (например, обращение самого бота к Proxmox API на порт `8006` или к локальному хосту) для предотвращения ложных алертов.
* **Защита хоста**: Контроль как исходящей активности LXC-контейнеров, так и самого хоста Proxmox VE.

### 2. 🔑 Мониторинг авторизаций и аудит в реальном времени
* **Динамический tailing логов прямо с хоста**: Асинхронный watcher для логов аутентификации (`auth.log`, `secure` или `journalctl`) каждого запущенного контейнера.
* **Контроль входов SSH**: Моментальные алерты об успешных входах (с указанием IP, имени пользователя и SSH-ключа) и неудачных попытках (определение брутфорс-атак).
* **Аудит консольных команд**: Отслеживание и логирование выполнения команд через `sudo` внутри LXC-контейнеров.

### 3. 🤖 Автоматизация Ansible & Динамический Инвентарь
* **Автоматическое развертывание ключей (One-Click Setup)**: Встроенный мастер настройки пользователя `ansible` с беспарольным доступом `sudo` во все активные LXC контейнеры (через `pct exec`) и на внешние VPS (по SSH) в один клик.
* **Генерация пары ключей ED25519**: При первом старте бот самостоятельно и безопасно генерирует новую пару ключей ED25519 для Ansible в папке плейбуков.
* **Динамический инвентарь `hosts.ini`**: Бот опрашивает запущенные контейнеры и внешние VPS, автоматически наполняя файл инвентаря, сохраняя при этом ваши кастомные группы, красивые названия хостов (`IP -> Имя`) и глобальные настройки `[all:vars]`.
* **Визуальный аудит готовности (Health Check)**: При входе в меню настройки бот в параллельном асинхронном режиме опрашивает все хосты на наличие пользователя и валидность SSH-ключа, отображая их готовность с помощью индикаторов `🟢` / `🔴`.
* **Устранение ошибок верификации**: В инвентарь автоматически внедряется параметр `ansible_ssh_common_args=-o StrictHostKeyChecking=no`, что позволяет запускать плейбуки без ручного подтверждения отпечатков хост-ключей.

### 4. 💾 Персистентность и надежность (Production-Ready Architecture)
* **Сохранение состояния в SQLite (Persistent State)**: Все данные о сессиях, временных нарушениях Hysteria 2 и троттлинге алертов атомарно записываются в SQLite базу данных `vpn_history.db`. В случае перезагрузки бота состояние мгновенно восстанавливается без потери контекста.
* **Нулевая потеря логов (Zero Log Loss)**: Механизм умного переподключения Watcher-а по SSH. В случае падения соединения с VPS бот автоматически считывает пропущенные логи за время простоя при помощи флага `--since "30 seconds ago"`, предотвращая пропуск алертов.
* **Отказоустойчивый Garbage Collector (Self-Healing)**: Временные блокировки записываются в базу данных с меткой времени окончания. Фоновый асинхронный воркер периодически очищает просроченные правила из `iptables` (как локально, так и удаленно), исключая "вечные" блокировки при перезагрузках бота.
* **Безопасность IPS для Администраторов**: Процессы `ansible`, `ansible-playbook` и `ssh` находятся в белом списке самозащиты. При запуске плейбуков IPS не блокирует целевые IP-адреса хостов, обеспечивая бесперебойную автоматизацию и защищая администратора от случайных блокировок.
* **Обход ограничений Telegram API (Media Message Edit)**: При навигации из объемных логов-документов бот автоматически пересоздает сообщения, исключая сбои Telegram API.

### 5. 📱 Интерактивный пульт управления (Telegram Dashboard)
* Управление нодами, виртуальными машинами и LXC-контейнерами через красивое кнопочное меню Telegram.
* Возможность запустить, остановить, перезагрузить или клонировать контейнер прямо из чата.
* Интерактивный просмотр последних логов авторизации и списка активных сетевых соединений контейнера по одной кнопке!

## ⚙️ Безопасная архитектура и чистота кода

* **Изолированное хранение секретов**: Все чувствительные файлы (`.env` с токенами, паролями и приватные SSH-ключи для внешнего мониторинга) перенесены в отдельную защищенную директорию `bot/config/`. Файл `.gitignore` гарантирует, что ваши приватные данные никогда случайно не попадут в публичный репозиторий.
* **100% Отсутствие хардкода**: Кодовая база бота полностью свободна от захардкоженных IP-адресов или токенов. Все настройки определяются динамически через окружение.
* **Полная автономность (Standalone-ready)**: Бот может работать как в связке с удаленными серверами (VPS), так и в изолированном «чистом» режиме локального гипервизора Proxmox VE.

## 🚀 Быстрая установка и интерактивная настройка

Мы создали удобный интеллектуальный установщик, который сделает всю рутинную работу за вас.

### 1. Подготовка папки на сервере Proxmox
Рекомендуется разместить проект в папке `/opt/pve-aegis`:
```bash
mkdir -p /opt/pve-aegis
# Скопируйте файлы вашего проекта (папку bot и .gitignore) в созданную директорию
```

### 2. Запуск интерактивного установщика
Запустите скрипт установки из-под пользователя `root` (это обязательно для управления iptables и мониторинга процессов):
```bash
cd /opt/pve-aegis/bot
chmod +x setup.sh
sudo ./setup.sh
```

В процессе установки скрипт:
1. Установит системные пакеты (python3-venv, pip и т.д.).
2. Создаст изолированное виртуальное окружение (`venv`) и установит зависимости.
3. Запустит **ИНТЕРАКТИВНЫЙ МАСТЕР НАСТРОЙКИ `.env`** прямо в терминале. Вы сможете легко заполнить токены Telegram, пароли к 3X-UI и параметры Proxmox (просто нажимайте Enter, чтобы сохранить значения по умолчанию!).
4. Создаст и активирует службу автозапуска `systemd`.

## 💻 Системные требования и сетевые настройки

Для того чтобы активная защита (IPS) могла перехватывать и блокировать трафик контейнеров, должны быть соблюдены два простых условия:

1. **Включение Сетевого экрана (Firewall) в Proxmox**:
   * Откройте Web-интерфейс Proxmox -> Выберите нужный LXC-контейнер -> вкладка **Network** (Сетевые карты).
   * Выберите сетевую карту `eth0` -> нажмите **Edit** -> Поставьте галочку **Firewall** (Сетевой экран) -> нажмите **OK**.
   *(Это создаст интерфейс фильтрации `fwbr` на хосте, позволяя `iptables` перехватывать трафик).*

2. **Автоматическая самодиагностика бота (Ядро Linux)**:
   * Наш бот полностью **самовосстанавливающийся**! При каждом старте службы он автоматически загружает модуль ядра `br_netfilter` и принудительно включает параметр `net.bridge.bridge-nf-call-iptables = 1`. 
   * **Настройка выживает при перезагрузках хоста!**

## 📝 Управление службой и просмотр логов

Служба бота добавлена в автозапуск и управляется стандартными системными командами:

* **Перезапустить службу**:
  ```bash
  sudo systemctl restart proxmox-lxc-bot.service
  ```
* **Проверить статус службы**:
  ```bash
  sudo systemctl status proxmox-lxc-bot.service
  ```
* **Просмотр логов в реальном времени**:
  ```bash
  sudo journalctl -u proxmox-lxc-bot -f --no-pager
  ```
